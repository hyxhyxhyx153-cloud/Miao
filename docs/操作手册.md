# 喵 Miao 操作手册

> 适用范围：Android 客户端、Fastify 服务端、Vue 管理后台与 Docker 依赖。
> 最后更新：2026-07-16

## 1. 默认账号与密码

### 1.1 管理后台

本项目**没有写死的默认管理员密码**，也不会从数据库中还原明文密码。管理员密码只以 bcrypt 哈希保存。

首次部署按下面的方式创建管理员：

1. 先在 Android 客户端注册账号，密码由部署人员自行设置（至少 8 位）。
2. 在服务器上把该账号提升为管理员：

```powershell
docker exec -it miao-postgres-1 psql -U miao -d miao -c "UPDATE users SET role='admin' WHERE email='你的管理员邮箱';"
```

3. 使用注册时设置的邮箱或用户名、密码登录管理后台。
4. 登录后在“管理员账号”页面可新增管理员或重置管理员密码。

如果数据库中已经有用户名为 `admin` 的账号，其密码也是创建该账号时设置的密码，并不是项目统一默认密码。忘记后应由另一个管理员重置；不要尝试从 `password_hash` 读取明文。

### 1.2 PostgreSQL 与 Redis 的开发默认值

`server/docker-compose.yml` 当前仅为本地开发提供以下默认值：

| 服务 | 默认值 |
|---|---|
| PostgreSQL 用户名 | `miao` |
| PostgreSQL 密码 | `miao-local-only` |
| PostgreSQL 数据库 | `miao` |
| Redis 密码 | 默认未设置 |

这些不是管理后台登录凭据。生产部署必须修改 PostgreSQL 密码并同步修改 `server/.env` 中的 `DATABASE_URL`；Redis 必须限制在本机或 Docker 内网，不能直接暴露到公网。

## 2. 启动顺序

在仓库根目录执行：

```powershell
docker compose -f server/docker-compose.yml up -d
npm --prefix server install
npm --prefix admin install
npm --prefix server run db:migrate
npm --prefix server run dev
```

另开终端启动管理后台：

```powershell
npm --prefix admin run dev
```

默认访问地址：

- API 健康检查：`http://localhost:3000/health`
- 管理后台开发站点：`http://localhost:5173`
- Android 模拟器访问宿主机 API：`http://10.0.2.2:3000/api/v1/`

## 3. 修改服务器地址

### 3.1 修改服务端监听端口与外部地址

编辑 `server/.env`：

```env
PORT=3000
LOCAL_BASE_URL=http://192.168.1.100:3000
CORS_ORIGINS=http://192.168.1.100:5173
TRUST_PROXY=false
```

- `PORT`：Fastify 实际监听端口。
- `LOCAL_BASE_URL`：客户端能够访问的 API 源地址，不要带 `/api/v1`。生成图片、上传文件和 APK 下载地址会使用它。
- `CORS_ORIGINS`：允许访问 API 的前端源，多个地址用逗号分隔，例如 `https://admin.example.com,http://192.168.1.100:5173`。
- `TRUST_PROXY`：只有在 API 位于自己可控的 Nginx/Caddy/隧道代理之后时才设为 `true`，否则管理后台 IP 白名单可能被伪造请求头绕过。

修改后重启服务端，并检查：

```powershell
Invoke-RestMethod http://localhost:3000/health
```

### 3.2 修改 Android 客户端 API 地址

地址在 `app/build.gradle.kts` 中通过 Gradle 属性注入，不需要修改 Kotlin 网络代码。可在项目根目录的 `gradle.properties`（不要提交含私有地址或密钥的本机配置）中加入：

```properties
# Android Studio 模拟器访问开发电脑
MIAO_EMULATOR_API_BASE_URL=http://10.0.2.2:3000/api/v1/

# 真机访问同一局域网中的服务器
MIAO_LAN_API_BASE_URL=http://192.168.1.100:3000/api/v1/

# 可选：强制所有构建使用同一个正式地址；设置后优先级最高
MIAO_API_BASE_URL=https://api.example.com/api/v1/
```

规则：

- 地址必须以 `http://` 或 `https://` 开头，并以 `/api/v1/` 结尾。
- 模拟器访问开发电脑使用 `10.0.2.2`，不能写 `localhost`。
- 真机使用服务器的局域网 IP；手机与服务器需互通。
- 正式 APK 推荐使用 HTTPS 域名设置 `MIAO_API_BASE_URL`。
- 修改后需要重新构建并安装 APK：`./gradlew.bat assembleDebug`。

### 3.3 修改管理后台连接地址

开发模式下，`admin/vite.config.js` 会把 `/api/v1` 和 `/health` 代理到：

```js
target: 'http://localhost:3000'
```

API 不在本机时，把两处 `target` 改为实际 API 源地址并重启 Vite。若要让局域网其他设备直接访问 Vite：

```powershell
npm --prefix admin run dev -- --host 0.0.0.0
```

生产环境中 `admin/src/http.js` 使用同源相对路径 `/api/v1`。推荐由 Nginx/Caddy 同时托管管理后台静态文件，并把 `/api/v1`、`/health` 反向代理到 `127.0.0.1:3000`，无需在前端写死 API IP。

### 3.4 配置模型中转 Base URL

在“后台 → API Key 管理”中填写模型服务商或中转站提供的 **Base URL**。自定义地址只会去除复制时带入的首尾空格，保存时不会添加 `/v1`，也不会增删末尾 `/` 或改写已有路径。

例如中转商给出的地址是：

```text
https://gateway.example.com/
```

就按上面的完整形式填写并保存，不要自行补成 `/v1`。如果中转商给出的是带路径的根地址，也应完整保留该路径。留空才会使用所选服务商的官方默认地址。

Base URL 是接口根地址，不是单个接口的完整地址。程序调用时，模型 SDK 会在该根地址后请求协议所需的接口，例如 OpenAI 兼容聊天的 `chat/completions`、模型列表的 `models` 或图片生成的 `images/generations`。如果自定义根地址返回的是管理网页或其他非 API 数据，OpenAI 兼容链路会仅在运行时再尝试一次 `/v1`；成功路径短时缓存在内存中，不会写回数据库或改变后台显示值。连接测试失败不影响先保存配置，但正式启用前应使用列表中的“测试”按钮验证中转商实际支持的协议和路径。

## 4. 内网映射与端口清单

### 4.1 推荐的生产映射

生产环境只映射反向代理端口：

| 端口/协议 | 是否映射到公网 | 用途 |
|---|---:|---|
| `443/TCP` | 必须 | HTTPS 管理后台、Android API、SSE 聊天流、图片与 APK 下载 |
| `80/TCP` | 可选 | HTTP 跳转 HTTPS、证书签发校验 |
| `3000/TCP` | 否（仅本机） | Fastify API，由 Nginx/Caddy 转发 |
| `5173/TCP` | 否 | Vite 开发服务器，生产不使用 |
| `5432/TCP` | **禁止公网映射** | PostgreSQL，只允许服务端或 Docker 内网访问 |
| `6379/TCP` | **禁止公网映射** | Redis，只允许服务端或 Docker 内网访问 |

Nginx/Caddy 的 SSE 代理读取超时建议不少于 300 秒，并关闭响应缓冲，否则聊天和图片生成进度可能被一次性延迟返回。

### 4.2 纯开发环境的内网穿透/端口转发

如果暂时没有 Nginx/Caddy，最少映射：

| 外部端口 → 内网目标 | 协议 | 必须性 | 用途 |
|---|---|---:|---|
| 自定义外部端口 → `服务器IP:3000` | TCP/HTTP | Android 必须 | API、SSE、上传/生成图片、APK 下载 |
| 自定义外部端口 → `服务器IP:5173` | TCP/HTTP | 仅需远程管理后台时 | Vite 管理后台开发站点 |

此模式下要同步修改：

- `LOCAL_BASE_URL` 为 API 的外部可访问地址；
- Android 的 `MIAO_API_BASE_URL`；
- `CORS_ORIGINS` 加入管理后台的外部源地址；
- Vite 用 `--host 0.0.0.0` 启动。

不要为了内网穿透而映射 `5432` 或 `6379`。数据库维护应通过 VPN、SSH 隧道或登录服务器后执行。

### 4.3 出站端口

服务器通常还需要允许以下**出站**连接，不需要做入站端口映射：

- `443/TCP`：OpenAI/Anthropic/其他模型 API、GPT Image 2、微信 ClawBot、OSS。
- `587/TCP` 或邮件服务商指定端口：SMTP 找回密码邮件。
- `53/UDP/TCP`：DNS 解析（通常由系统网络自动提供）。

## 5. 生产反向代理要点

示例结构：

```nginx
server {
    listen 443 ssl;
    server_name miao.example.com;

    root /opt/miao/admin/dist;
    index index.html;

    location /api/v1/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_buffering off;
        proxy_read_timeout 300s;
    }

    location /health {
        proxy_pass http://127.0.0.1:3000/health;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

构建管理后台：

```powershell
npm --prefix admin run build
```

## 7. 安全检查

- 不要提交 `server/.env`、`local.properties`、API Key、JWT 密钥或数据库生产密码。
- 生产环境替换 `JWT_SECRET`、`JWT_REFRESH_SECRET`、`DATA_ENCRYPTION_KEY` 和 Docker 数据库默认密码。
- GPT Image、聊天模型等 API Key 应在管理后台配置；自定义 Base URL 必须严格按服务商或中转商给出的接口根地址填写，保存值不会被添加 `/v1` 或改写路径；非 API 根页面仅会触发不落库的运行时兼容重试。
- 管理后台开启 IP 白名单时，先确认受信任反向代理和 `TRUST_PROXY` 配置正确。
- 首个管理员创建完成后立即使用强密码，并至少保留两个可用管理员账号以便应急恢复。
