# 喵 · UI 设计系统

> 版本：v2.0 · Apple Liquid Glass 方向 · 2026-07-12

## 设计目标

喵的界面使用“安静内容层 + 浮动功能层”的结构。内容负责清晰阅读，Liquid Glass 只用于导航、工具栏、输入区、浮层和需要表达层级的控制面。玻璃不是透明白卡的同义词，也不能模糊文字、图片或二维码。

设计关键词：克制、通透、亲近、精确。

- 克制：取消紫粉满屏渐变和彩虹状态色，品牌只保留一个蓝紫强调色。
- 通透：材质由半透明填充、边缘高光、环境色和柔和阴影共同形成。
- 亲近：猫系元素集中在品牌标记、头像和少量空状态，不使用 Emoji 代替所有功能图标。
- 精确：数据使用等宽数字，文字层级、触控面积、焦点态和状态反馈均有明确规范。

## 层级原则

1. `Canvas`：暖灰或深墨背景，可放低饱和径向环境光。
2. `Content`：列表、消息和数据表，保持较高不透明度与稳定对比。
3. `Glass controls`：Tab bar、Top bar、Sidebar、输入胶囊、筛选工具条和 Bottom Sheet。
4. `Transient`：菜单、Dialog、Drawer 和状态提示，位于最高视觉层。

禁止对包含文字、二维码或关键图片的整体容器使用内容模糊。Android 不具备可靠背板模糊时，使用半透明填充和内高光降级；Web 不支持 `backdrop-filter` 时使用高不透明背景。

基础间距：`4 / 8 / 12 / 16 / 24 / 32`。Android 触控目标不小于 48dp；Web 主要操作不小于 44px。圆角需要按层级变化，禁止每个容器使用同一半径。

## 字体

- Android：系统 Sans，不分发受限的 Apple 字体文件。
- Admin：`-apple-system, BlinkMacSystemFont, "SF Pro Display", "SF Pro Text", "PingFang SC", "Segoe UI", sans-serif`。
- 页面大标题 28–34，Section 20，正文 15–17，Footnote 13。
- 辅助文字使用正常体；斜体仅用于动作描述等真正需要语气的内容。
- 配额、Token、耗时、日期和表格数字启用 tabular numbers。

## App 页面规范

### 登录与注册

使用低饱和环境光背景和单个玻璃登录面板。Logo、标题、说明、输入区和主操作形成一条清晰阅读路径。键盘弹出后主要操作必须保持可见，不使用饱和满屏渐变。

### 主导航

底栏是浮于内容上方的强玻璃胶囊，遵守系统安全区。选中态使用单一 Accent 的浅色容器，不再铺满页面宽度。Top bar 保持轻量，仅显示当前任务所需信息。

### 会话与记忆

采用 iOS inset grouped list：多个条目共享一个稳定内容面，条目间以 hairline 分隔。头像使用 squircle，操作放入滑动或上下文菜单，避免每条记录独立成卡和常驻多个按钮。

### 聊天

聊天页单独采用微信式信息架构，不套用其他页面的玻璃内容卡：

- 背景为微信式浅灰 `#EDEDED`，暗色模式使用同层级深灰。
- AI 消息左对齐：squircle 头像 + 白色气泡 + 左侧小尖角。
- 用户消息右对齐：微信绿 `#95EC69` 气泡 + 右侧小尖角，正文使用深色。
- 时间与系统/动作提示在中间以弱灰小字呈现。
- 顶栏保持简洁：返回、会话名、紧凑模型/配额状态和更多操作。
- 底部工具栏顺序为语音、输入框、表情、加号；输入内容后显示发送操作。
- 图片、表情、Markdown、错误重试、停止生成和长按菜单保留原功能，但视觉遵循微信消息节奏。
- 错误、重试和配额耗尽必须可见，不能只依赖红色表达。

### 设置与资料

资料 Hero 只出现一次；设置采用分组列表、Section 标题和 squircle 图标。危险操作独立成组。进度条与状态使用语义色，不使用紫粉渐变。

### 微信二维码

二维码始终置于纯白不透明平面，保留 quiet zone，不应用玻璃、透明度或模糊。扫码、确认、成功/过期使用明确步骤状态。

## Admin 页面规范

### Shell

桌面端使用浮动玻璃 Sidebar 和 command header，内容宽度不超过 1600px。窄屏 Sidebar 变为可关闭抽屉；页面高度使用 `100dvh`。导航必须具备当前页、键盘焦点和收起状态。

### Dashboard

首屏只突出少量关键 KPI，其余指标降为次级条目。图表统一单一 Accent、淡网格和真实空状态。禁止通过彩虹卡片制造层级。

### 数据工作区

表格和长表单使用高不透明内容面，保证密度与可读性；筛选条、分页、快捷操作可使用玻璃。Row hover、focus、selected 和 error 必须可区分，危险操作与普通操作分离。

### 登录

宽屏采用品牌说明与登录表单双栏玻璃面板，窄屏合为单栏。错误使用表单内联 Alert，不弹浏览器原生提示；加载状态禁用重复提交。

## 动效与无障碍

- 控件反馈 160–220ms，页面切换约 260ms，使用 transform/opacity。
- 按压缩放不低于 0.975，避免夸张弹跳。
- 系统开启 `prefers-reduced-motion` 或减少动画时关闭非必要位移和环境动画。
- 文本与背景保持可读对比；不要用透明度过低的灰字承载关键状态。
- 支持大字体、长中文、长模型 ID、99+ 未读、键盘导航和屏幕阅读器描述。
