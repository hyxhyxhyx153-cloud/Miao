# 喵 · 后台管理端文档

> 版本：v2.0 · 日期：2026-07-12

---

## 一、概述

后台管理端（Web Admin）是供运营 / 管理员使用的 Web 控制台，管理 API Key、用户账户、模型配置、系统监控等，用户 App 端不可见。

- 技术栈：Vue 3 + Vite + Element Plus + Pinia
- 访问路径（示例）：`https://admin.miao.example.com`
- 鉴权：独立 Admin 账号，JWT（role=admin），与用户 Token 隔离

## 三、登录页

- 邮箱 + 密码登录。
- 登录成功后跳转仪表盘，JWT 存入 sessionStorage（关闭浏览器即失效）。
- 连续 5 次错误密码触发 15 分钟冷却锁定。

## 六、API Key 管理

**这是后台最核心的模块**。所有 AI 模型的 API Key 统一在此管理，用户 App 不接触任何 Key。

### 6.1 Key 列表

表格列：

| 列名 | 说明 |
|---|---|
| 提供商 | Anthropic / DeepSeek / Qwen / 智谱 / OpenAI |
| Key 预览 | 只显示前 8 位 + `****`，点击眼睛图标展示完整（需再次输入管理员密码确认） |
| Base URL | 当前生效的 API 地址 |
| 状态 | 启用 / 禁用 |
| 连通性 | 上次测试时间 + 结果（绿勾 / 红叉） |
| 今日用量 | Token 消耗（今日） |
| 备注 | 管理员填写的说明 |
| 操作 | 编辑 / 测试 / 禁用 / 删除 |

### 6.2 新增 / 编辑 Key

表单字段：

- 提供商（下拉选择）
- API Key（文本框，输入时隐藏，提交后加密存储）
- Base URL（可选，留空使用官方默认地址，支持填写中转代理地址）
- 备注（可选，如"主 Key"、"备用 Key"）

一个提供商可配置多个 Key（用于轮转），系统按顺序轮转使用，某个 Key 失败后自动切换到下一个。

### 6.3 Key 轮转规则

- 默认策略：**顺序轮转**（每次请求使用下一个可用 Key）。
- 失败策略：单个 Key 连续失败 3 次后自动标记为"暂时禁用"，10 分钟后恢复。
- 管理员可手动设置某个 Key 为主用（其余为备用）。

### 6.4 连通性测试

- 点击"测试"按钮：服务器向对应 AI Provider 发送一个最小请求（如"Hi"），展示响应时间和结果。
- 也可在列表页点击"批量测试全部"，一键验证所有 Key。

### 6.5 用量监控

- 折线图：各 Key 的 Token 消耗趋势（近 7 天，按日/小时切换）。
- 可设置 Token 消耗告警阈值（超出后发送邮件通知管理员）。

```
│                                          │
│  [取消]                      [保存上传]  │
└─────────────────────────────────────────┘
```

**AI 自动识别（单张）：**

点击"识别"按钮后：
1. 服务器以该图片调用所选 Vision 模型（使用后台已配置的对应 API Key）。
2. Prompt 为固定模板（见 8.5 节），要求模型返回 `emotion_tag`、`description`、`scene_keywords` JSON。
3. 识别完成后结果自动填入表单对应字段，管理员可预览并修改后再保存。
4. 全程约 3–8 秒，按钮变为 loading 状态，完成后恢复。

**图片格式要求：**
- 支持：WebP（推荐）、PNG、GIF（仅取第一帧）、JPG
- 最大单文件：5MB
- 推荐尺寸：至少 160×160px，最大不限
- 上传后服务器自动生成 72×72 缩略图（WebP 格式）

### 8.5 AI 识别 Prompt 模板

服务器调用 Vision 模型时使用固定 Prompt，管理员可在系统设置中覆盖：

```
你是一个表情包情绪分析专家。分析这张猫娘主题表情包图片，返回以下 JSON（不要返回其他任何内容）：

{
  "emotion_tag": "从以下选项中选一个最匹配的: happy/excited/curious/shy/embarrassed/caring/gentle/playful/thinking/surprised/sad/nervous/proud/sleepy/angry",
  "confidence": 0.0到1.0之间的置信度,
  "alternative_emotion": "第二候选情绪（置信度差距<20%时填写，否则填null）",
  "description": "一句话描述图片内容，15字以内，如：猫娘开心大笑，露出虎牙",
  "scene_keywords": ["关键词1", "关键词2", "关键词3"]
}
```

- Prompt 使用服务器环境变量 `EMOJI_RECOGNIZE_PROMPT` 覆盖。
- 当 `confidence < 0.6` 或 `alternative_emotion` 非 null 时，该图标记为"待确认"状态。
- 返回的 JSON 经服务器严格校验（Zod schema），解析失败时标记为"识别失败"并记录原始响应供排查。

## 九、人格管理（内置人格）

管理员在此维护全平台所有用户都能看到的内置人格（`is_builtin=true`）。

表单字段：名称、描述（一句话）、系统提示词全文、是否启用。

内置人格用户只读，不可删除，但可以复制为自定义人格。

## 十、系统监控

### 9.1 实时请求日志

表格列：时间、用户名、会话 ID、模型、请求 Token 数、响应 Token 数、耗时（ms）、状态码。

- 支持按时间、用户、模型、状态筛选。
- 仅保留近 7 天日志（历史聚合统计另行保存）。

### 9.2 错误日志

- 展示 AI Provider 返回的非 200 响应，包含：时间、提供商、错误码、错误信息、Key 预览（前 8 位）。
- 帮助管理员快速发现 Key 失效、限流、服务异常等问题。

## 十一、系统设置

### 10.1 全局配额设置

| 设置项 | 说明 |
|---|---|
| 新用户默认每日配额 | 默认 50 条，可调整 |
| 配额重置时间 | UTC 00:00（固定） |
| 配额耗尽提示文案 | 可自定义，App 内展示 |

### 10.2 微信绑定管理

后台管理端**不配置任何微信凭据**，凭据由用户扫码获取后自动保存在服务器，管理员只负责运维管理。

**绑定列表视图**

| 列名 | 说明 |
|---|---|
| 用户名 | 关联的喵账号 |
| 绑定时间 | 完成扫码的时间 |
| Worker 状态 | 运行中 / 已停止 / 错误（带最近错误信息） |
| 最后活跃 | 最近一次收到微信消息的时间 |
| 今日消息量 | 该绑定账号今日处理的微信消息条数 |
| 操作 | 重启 Worker / 强制解绑 |

**Worker 重启**：停止当前 Worker → 重新 notifystart → 重建长轮询循环，不需要用户重新扫码。

**强制解绑**：调用 Clawbot `channel_reset` 接口 → 停止 Worker → 将 `wechat_bindings.is_active` 设为 false，用户需重新扫码才能恢复。

### 10.3 管理员账号管理

- 查看所有 Admin 账号列表。
- 新增 / 删除 Admin 账号。
- 强制修改某个 Admin 密码。
- 至少保留 1 个 Admin 账号，不允许全部删除。

## 十二、安全规范

- **Key 展示**：API Key 完整内容仅在二次输入管理员密码后可见，操作记录留存审计日志。
- **审计日志**：所有 Admin 操作（封禁用户、修改 Key、改配额等）记录操作人 + 时间 + 操作内容，不可删除。
- **IP 白名单**（可选）：可配置只允许特定 IP 段访问后台管理端。
- **会话超时**：管理员 30 分钟无操作自动退出。
