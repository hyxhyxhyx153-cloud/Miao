```
喵 App
├── 登录 / 注册页
├── 会话列表页
├── 聊天页
│   ├── 消息气泡（文字 / 图片 / 表情包）
```

## 二、登录 / 注册

### 2.1 登录页

- 界面：全屏喵渐变背景 + 液态玻璃输入卡片，居中布局。
- 字段：邮箱 + 密码。
- "记住我"复选框：勾选后 Refresh Token 本地持久化，下次自动登录。
- "忘记密码"：跳转找回密码页（输入邮箱，发送重置链接）。
- "还没有账号？立即注册"：跳转注册页。

### 2.2 注册页

- 字段：用户名（唯一，2–32位字母数字下划线）、邮箱（唯一）、密码（≥8位）、确认密码。
- 注册成功后自动登录，跳转会话列表页。

### 2.3 鉴权机制

- 登录成功后服务器返回 `access_token`（有效期 2h）和 `refresh_token`（有效期 30d）。
- `access_token` 存入内存；`refresh_token` 存入 `EncryptedSharedPreferences`。
- 每次请求自动附加 `Authorization: Bearer <access_token>`。
- `access_token` 过期时，App 无感刷新（拦截器自动调用 `/auth/refresh`）。
- 退出登录时调用 `/auth/logout`，本地清除所有 Token 和缓存。

## 三、会话列表

### 3.1 展示

- 从服务器拉取会话列表，本地 Room 做离线缓存，下拉刷新同步最新数据。
- 每条会话卡片显示：
  - 猫娘头像（绑定人格的头像，圆形液态玻璃描边）
  - 会话名称（默认为模型名，可自定义）
  - 最后一条消息摘要（最多 2 行，超出省略）
  - 最后消息时间（今天显示时分，更早显示月日）
  - 未读角标（红底白字）

### 3.2 操作

- **长按**：弹出操作菜单（置顶 / 标记已读 / 删除）。
- **左滑**：快捷"删除"按钮，二次确认后调用服务器删除接口。
- **右上角 "+" 按钮**：进入新建会话流程。
- **搜索栏**（下滑展开）：按会话名 / 消息内容在本地缓存内模糊搜索，在线时同步服务器搜索。

## 四、聊天页

### 4.1 消息气泡

| 角色 | 气泡位置 | 样式 |
|---|---|---|
| 用户 | 右侧 | 微信式绿色气泡（浅色 `#95EC69`），带小尾巴 |
| AI（猫娘） | 左侧 | 微信式白色气泡，带小尾巴 |
| 系统提示 | 居中 | 小号灰色文字，无背景 |

- AI 消息支持 **Markdown 渲染**（标题、粗体、代码块含语法高亮、列表、分割线）。
- 代码块右上角有"复制"按钮。
- 消息间距 > 5 分钟时，显示时间分隔条。
- AI 消息中可夹带颜文字（如 `(´▽｀)`、`(｡>ω<｡)`），直接渲染在文字中，不另行处理。

---

### 4.2 思考状态框（AI 思考中）

用户发送消息后、AI 开始流式输出前，在 AI 气泡区域显示**思考状态框**：

```
┌─────────────────────────────┐
│  🐾  喵喵正在想想...         │  ← 液态玻璃卡片，带跳动爪印 Loading
│  ▸ 检索到 3 条相关记忆       │  ← 折叠展示，记忆检索完成后出现
└─────────────────────────────┘
```

- 爪印 Loading：3 个爪印图标依次出现并消失，颜色为喵紫，循环播放。
- 记忆检索结果：以折叠行形式显示"检索到 N 条相关记忆"，可点击展开查看具体内容。
- AI 开始输出第一个 token 后，思考框淡出，气泡正式开始追加内容。

### 4.4 动作提示条（仿微信撤回提示）

AI 在对话中可执行"动作"，动作以**类微信撤回消息**的居中小提示条形式出现在消息列表中，不占气泡位置：

```
         ·  喵喵 轻轻摇了摇尾巴  ·
         ·  喵喵 悄悄往主人身边凑了凑  ·
         ·  喵喵 发出了满足的呼噜声～  ·
         ·  喵喵 把头埋进爪子里  ·
```

**样式规格：**
- 字号：11sp，颜色：次级色 + 50% 透明，斜体。
- 左右各有装饰点（·），行高压缩，视觉上轻于正文消息。
- 淡入动效，持续 200ms。

**触发方式：**
服务器 SSE 的 `done` 事件中，除 `emotion` 外，同时可携带 `action` 字段（字符串），App 收到后在消息列表末尾插入动作提示条。

**常见动作文本（示例，服务器动态生成）：**
- 撒娇类：`喵喵 蹭了蹭主人的手`、`喵喵 发出了轻柔的喵喵声`
- 害羞类：`喵喵 把耳朵压低了`、`喵喵 悄悄藏起了尾巴`
- 开心类：`喵喵 的尾巴开始左右摇摆`、`喵喵 高兴地转了个圈`
- 思考类：`喵喵 歪了歪脑袋`、`喵喵 用爪子挠了挠下巴`
- 困倦类：`喵喵 打了个小哈欠`、`喵喵 的眼睛眯成了一条缝`

### 4.5 表情包系统（App 专属）

表情包仅在喵 App 内发送，**不会通过微信 Clawbot 通道发出**。

#### 4.5.1 表情包库架构

喵 App 内置一套猫娘主题表情包库，基于 **MaiBot 表情包情绪标签体系**设计，所有表情包按 `emotion_tag` 分类存储在服务器，App 启动时增量同步到本地缓存（Room + 文件缓存）。

**表情包元数据结构：**
```json
{
  "id": "uuid",
  "filename": "happy_001.webp",
  "emotion_tag": "happy",
  "description": "猫娘开心大笑，露出虎牙",
  "scene_keywords": ["开心", "大笑", "高兴"],
  "url": "https://cdn.miao.example.com/emojis/happy_001.webp",
  "thumb_url": "https://cdn.miao.example.com/emojis/thumbs/happy_001.webp"
}
```

**情绪分类（emotion_tag）与初始图包数量：**

| emotion_tag | 中文 | 初始数量 |
|---|---|---|
| happy | 开心 | 8 |
| excited | 兴奋 | 6 |
| curious | 好奇 | 6 |
| shy | 害羞 | 8 |
| embarrassed | 不好意思 | 6 |
| caring | 关心 | 6 |
| playful | 调皮 | 8 |
| thinking | 思考 | 6 |
| surprised | 惊讶 | 6 |
| sad | 难过 | 4 |
| sleepy | 困倦 | 4 |
| proud | 自豪 | 4 |

#### 4.5.2 AI 自动触发表情包

AI 回复生成完毕后，服务器根据 `emotion` 和内容语义，以**约 30% 的概率**自动触发表情包发送：
- 情绪强烈时（excited、shy、surprised）触发率提高到 50%。
- 多轮连续对话中同一情绪不连续触发（避免刷屏）。
- 表情包作为**独立消息**附在文字回复气泡之后，间隔 600ms 出现（有轻弹入场动效）。

#### 4.5.3 用户手动发送表情包

输入区右侧新增"表情包"按钮（🐾图标），点击展开表情包面板：

```
┌─────────────────────────────────────┐
│  [情绪分类 Tab 横向滚动]              │
│  开心 害羞 好奇 调皮 惊讶 思考 ...   │
├─────────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌──────┐        │
│  │表情包│ │表情包│ │表情包│  ← 3列网格│
│  └──────┘ └──────┘ └──────┘        │
│  ┌──────┐ ┌──────┐ ┌──────┐        │
│  │表情包│ │表情包│ │表情包│        │
│  └──────┘ └──────┘ └──────┘        │
└─────────────────────────────────────┘
```

- 面板高度：240dp，可上下滚动。
- 顶部 Tab：按 emotion_tag 分类，横向滚动，选中 Tab 高亮。
- 每格：72×72dp，圆角 8dp，展示缩略图，点击立即发送。
- 长按表情包：预览大图（260dp 宽弹窗，带描述文字）。

#### 4.5.4 表情包消息气泡样式

- 无文字气泡背景，直接展示图片，圆角 12dp。
- 图片尺寸：160dp（正方形），实际图片等比缩放填充。
- 发送方（用户）：右对齐；AI：左对齐，带头像。

### 4.6 图片上传

#### 4.6.1 用户发送图片

输入区"+"扩展面板 → **图片**（从 v1.0 起正式可用）：

- 触发系统相册选择器（Android 系统 Photo Picker）。
- 支持同时选择最多 **9 张**图片（仿微信上限）。
- 选择后进入图片预览页：
  - 横向滚动预览已选图片，左上角数字标注序号。
  - 每张图片右上角有"×"可单独删除。
  - 底部：输入框（可附带文字描述）+ 发送按钮。
- 发送逻辑：
  1. 图片上传到服务器 OSS（进度条显示在气泡处，带百分比）。
  2. 上传完成后，图片 URL 和文字描述一并发送给 AI。
  3. AI 回复（支持视觉分析能力的模型，如 Claude、GPT-4o）可理解图片内容。

- **气泡样式**：
  - 单图：最大 200dp 宽，等比显示，圆角 12dp。
  - 多图（2–4张）：2×2 宫格。
  - 多图（5–9张）：3列宫格，最后一格若为奇数则居左。
  - 图片点击：全屏查看，支持双指缩放，左右滑动切换。

#### 4.6.2 AI 识别图片

- 发送图片时，服务器将图片 URL 注入多模态 prompt（`image_url` 类型的消息 content）。
- 仅当所选模型支持 Vision 时启用此功能；不支持的模型发送图片后显示系统提示"当前模型不支持图片识别，已发送图片描述"。
- 模型是否支持 Vision 由后台管理端配置（`vision_enabled` 字段）。

### 4.7 发送消息与流式接收

1. 用户点击发送 → App 调用 `POST /api/v1/conversations/:id/chat`。
2. 服务器从数据库取上下文 → 检索相关记忆（向量语义搜索）→ 构造 Prompt → 调用对应 AI Provider → SSE 流透传。
3. App 通过 SSE 逐步接收事件，实时追加到气泡。

**SSE 事件扩展格式：**

```
data: {"type":"thinking_start"}                          ← 显示思考框
data: {"type":"memory","count":3,"items":[...]}          ← 记忆检索结果
data: {"type":"delta","content":"喵～"}                  ← 文字流
data: {"type":"done","emotion":"shy","action":"喵喵 悄悄往主人身边凑了凑","usage":{...}}
data: {"type":"emoji","id":"uuid","url":"..."}           ← 可选，触发表情包
data: [DONE]
```

---

### 4.8 停止生成

- 输出期间发送按钮变为"停止"（方形图标）。
- 点击停止：断开 SSE，已输出内容保留，`action` 和 `emoji` 事件被取消。

---

### 4.9 消息长按菜单

- 复制文本
- 重新生成（仅 AI 消息，调用 regenerate 接口）
- 删除消息（二次确认后调用服务器删除接口）
- **添加到记忆**（将选中消息内容直接保存为记忆片段）
- 撤回（仅用户自己发的消息，2 分钟内有效，撤回后在消息位置显示"你撤回了一条消息"提示条）

```
┌─────────────────────────────────────────┐
│ [+]  [ 跟喵说点什么吧～           ]  🐾 [→]│
└─────────────────────────────────────────┘
```

### 4.11 配额提示

- 聊天页顶部导航栏下方显示今日剩余配额（小标签，如"剩余 32 条"）。
- 配额剩余 ≤5 时标签变橙色警告。
- 配额耗尽时，输入框禁用，显示喵喵叹气的图片 + "今日消息已用完喵，明天再来吧～"。

### 4.12 会话模型与双方头像

- 聊天页左上角返回按钮旁提供“⋯”会话设置入口。
- 设置面板统一展示当前会话可用模型、我的头像和 AI 头像，不再把模型选择分散到标题或其他入口。
- 模型列表来自服务端，显示图片理解能力和停用状态；切换成功后从下一条消息开始生效，并同步写入服务端与 Room。
- 我的头像和 AI 头像均可从系统相册选择、真实上传，并支持恢复默认；设置只影响当前会话，所有历史与新消息气泡会立即使用新头像。
- 微信专属会话首次创建时采用用户的默认模型；创建后可在 App 会话设置中调整，之后不会被全局默认值覆盖。

## 五、记忆管理页

路径：底部导航"记忆"Tab，或我的页 → 记忆管理

### 5.1 记忆列表

- 卡片列表，每条记忆显示：内容摘要（2 行省略）、来源标签（手动 / 自动）、创建时间。
- 支持关键词搜索（服务器语义搜索）。
- 右上角"+ 添加"按钮。

### 5.2 添加记忆

- 底部弹出输入卡（Bottom Sheet）。
- 输入框：记忆内容（无字数上限，建议 1–200 字）。
- 确认添加 → 调用 `POST /api/v1/memories` → 服务器生成向量 embedding 存储。

### 5.3 编辑 / 删除记忆

- 长按记忆卡片：弹出"编辑 / 删除"菜单。
- 删除需二次确认弹窗。

### 5.4 记忆在对话中的作用

- 每次发送消息时，服务器在后台自动进行语义检索，找出最相关的 3–5 条记忆，以系统消息形式注入对话上下文。
- AI 不会主动"报告"记忆的存在，而是自然地运用这些信息。
- 注入的记忆条目在聊天页消息列表顶部以折叠形式展示（可展开查看），让用户了解本次对话参考了哪些记忆。

## 六、新建会话

底部弹出引导卡（Bottom Sheet），分两步：

**第一步：选择 AI 模型**

- 横向滑动卡片，每张显示：品牌 Logo、模型名称、一句话描述。
- 模型是否可用由后台配置控制（管理员可开关），不可用的模型卡片显示"暂不可用"遮罩。
- 支持：Claude（Anthropic）、DeepSeek、Qwen（通义千问）、GLM（智谱）、OpenAI（GPT）。

**第二步：选择 / 创建人格**

- 内置选项：喵喵（猫娘）、知识助手、写作助手、代码助手。
- "自定义"：输入任意系统提示词并命名，保存到服务器。

## 七、微信绑定（openclaw-weixin）

路径：我的 → 微信绑定

### 7.1 功能描述

用户通过微信扫描 Clawbot 二维码完成绑定，之后可直接在微信中与 AI 猫娘对话。消息记录、记忆与 App 内完全同步。整个过程**无需任何配置**，扫码即用。

每个喵账号只能绑定一个微信账号。

### 7.2 绑定流程

```
用户点击"绑定微信"
    │
    ▼
App 调用喵服务器 → 服务器调 Clawbot API
POST /api/v1/wechat/qrcode（无需参数）
    │
    ▼
返回 { qrcode_url, qrcode }
App 展示二维码（来自 qrcode_url）
    │
    ▼
服务器持续轮询 Clawbot
POST /api/v1/wechat/qrcode/status { qrcode }
    │
    ├── "wait"      → App 显示"等待扫描..."
    ├── "scaned"    → App 显示"已扫描，请在微信中点击确认"
    ├── "expired"   → App 显示过期遮罩 + 刷新按钮（重新请求二维码）
    └── "confirmed" →
              credentials.bot_token + credentials.baseurl
              服务器保存到 wechat_bindings 表（加密存储）
              启动 WechatBotWorker（长轮询 getupdates）
              App 显示绑定成功动画
```

### 7.3 消息通道工作方式

绑定成功后，喵服务器在后台持续运行一个 Worker，通过 openclaw-weixin 的 iLink Bot API 与微信保持长连接：

```
微信用户发消息
    ↓
WechatBotWorker 的 getupdates 收到 WeixinMessage
    ↓
查找对应喵用户 → 检索记忆 → 构造上下文
    ↓
调用 AI Provider（使用后台统一 API Key）
    ↓
sendmessage 回复微信（附带 context_token）
    ↓
消息写入数据库（source='wechat'），App 内可查看
```

### 7.4 已绑定状态

- 我的页"微信绑定"行显示"已绑定 ✅"。
- 进入页面可查看：绑定时间、Worker 运行状态。
- "解除绑定"按钮：二次确认 → 服务器调用 `channel_reset` → 停止 Worker → 清除绑定记录。解绑后该微信账号不再接收消息，历史记录保留。

### 7.5 微信消息在 App 内的展示

- 绑定后，系统自动创建一个专属的"微信"会话（或复用已有的），来自微信的消息出现在该会话中。
- 微信消息气泡左侧显示微信小图标标识来源。
- App 内可查看完整的微信对话历史，但不能从 App 端向该会话发消息（只能通过微信发送）。

## 九、数据云同步策略

| 数据类型 | 同步时机 | 冲突处理 |
|---|---|---|
| 会话列表 | App 前台激活时拉取；创建/删除会话时立即推送 | 服务器为主，本地缓存覆盖 |
| 消息记录 | 每次发送/接收立即同步；进入聊天页时拉取最新 100 条 | 以 message_id 去重 |
| 记忆 | 增删改立即推送服务器 | 服务器为主 |
| 用户设置 | 修改后立即推送；登录时拉取 | 服务器为主 |
| 离线状态 | 本地操作写入 Room 并标记 `is_synced=false`；恢复网络后自动重试 | 服务器最终一致 |

## 十、错误处理

| 错误类型 | 用户侧展示 |
|---|---|
| 未登录 / Token 过期 | 静默刷新 Token；刷新失败则跳转登录页 |
| 网络超时 | 消息气泡变红，"请求超时，点击重试" |
| 配额不足（403） | "今日消息已用完喵，明天再来吧～" |
| 模型不可用（503） | "这个模型暂时休息了，要换一个试试吗？" |
| API 返回错误 | 显示简化错误描述，可展开查看原始错误 |
| 上下文超长 | 服务器自动截断，系统消息提示"已自动清理早期对话" |
| 微信二维码过期 | 叠加遮罩，显示刷新按钮 |
