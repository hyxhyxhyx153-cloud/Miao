## 关键修复记录

- API Key 新增、修改、删除会同步持久化配置，服务重启后仍生效。
- App 所有聊天失败会保存为可见错误消息，可从消息菜单重试。
- 配额 GET、资料 GET 和聊天发送均执行跨日重置，App 每次进入聊天重新同步。
- 消息 clientId 唯一约束为 `(conversation_id, client_id)`，不会跨会话覆盖。
- 客户端停止生成会关闭 SSE，服务端通过 AbortSignal 终止 Provider 流，不再后台保存完整回答。
- 微信 Worker 仅在当前批次获得正常回复或持久化错误回复后推进游标；重启可恢复中断消息。
- ClawBot 二维码使用官方接口返回的二维码内容，不再使用演示或伪二维码。
- 后台路由取消阻塞式退场动画，并将开发代理收紧至 `/api/v1`，`/api-keys` 等页面不再被误转发或需要手动刷新。
- 会话模型切换由服务端模型注册表校验并推导 Provider；停用模型会被拒绝，双方头像只接受 HTTP/HTTPS 地址或显式清除。

## 部署注意事项

- 生产环境必须设置 `JWT_SECRET`、`JWT_REFRESH_SECRET` 和 `DATA_ENCRYPTION_KEY`。
- 邮件重置与 Token 告警需要配置 SMTP；对象存储可配置 OSS，未配置时使用本地上传目录。
- `CORS_ORIGINS`、`TRUST_PROXY` 和管理端 `ip_whitelist` 应按实际部署网络填写。
- 当前 Debug APK 的 API 地址由 `BuildConfig.API_BASE_URL` 决定，真机发布前应指向可访问的正式 HTTPS API。
